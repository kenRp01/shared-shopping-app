# ShareShopi 共有設定改善FW

## 目的

共有リストを使い始めるまでの手間を減らす。設定画面へ移動して複数項目を読む流れではなく、リスト画面から最短で共有できる体験にする。

対象案:

- 案2: リスト画面から直接メンバー追加
- 案3: 共有リンク方式
- 案5: QRコード共有

## 前提

- 個人利用はログイン不要のまま残す。
- ユーザー間共有はログイン必須にする。
- 共有リンクまたはQRを受け取った人も、編集参加するにはログインが必要。
- 公開リンク閲覧とは別物として扱う。公開リンクは閲覧専用、招待リンクはログイン後に編集メンバーへ追加する。
- 無料運用を優先し、外部の招待メール送信は必須にしない。

## 案2: リスト画面から直接メンバー追加

### UI

- `共有` タブを開いているとき、タブ列または商品一覧上部に「人を追加」ボタンを表示する。
- ボタン押下で小さなモーダルまたは下部シートを開く。
- 入力はメールアドレスだけにする。
- 成功後は「共有しました」と表示し、メンバー一覧の詳細表示は設定画面に任せる。

### 処理

- 既存の `addListMember(listId, email, user)` を利用する。
- ゲストの場合はログイン誘導を表示する。
- 所有者以外はメンバー追加不可にする。

### 受け入れ条件

- 共有リスト画面から設定画面を開かずにメンバー追加できる。
- 未ログイン時は「共有にはログインが必要」と表示される。
- 登録済みユーザーのメールだけ追加できる。
- 追加後、相手のアカウントで同じリストが表示される。

## 案3: 共有リンク方式

### UI

- 共有シート内に「招待リンクをコピー」を表示する。
- コピー後は「コピーしました」と短く表示する。
- リンクの説明は最小限にし、「ログインすると編集メンバーに参加できます」程度にする。

### データ

- `shopping_lists` に招待用トークンを追加する案がシンプル。
- 候補カラム:
  - `invite_token`
  - `invite_token_created_at`
  - `invite_token_enabled`
- 公開閲覧用の `public_token` とは分ける。

### 画面

- `/invite/[token]` を追加する。
- 未ログインならログイン画面へ誘導し、ログイン後に再度 `/invite/[token]` へ戻す。
- ログイン済みなら `shopping_list_members` に `editor` として追加する。
- 参加後は対象リストへ遷移する。

### 受け入れ条件

- 所有者が招待リンクをコピーできる。
- 招待リンクを開いたログインユーザーが共有メンバーになる。
- すでに参加済みなら重複追加せず対象リストを開く。
- 無効なトークンでは参加できない。
- 公開リンク閲覧URLとは権限が混ざらない。

## 案5: QRコード共有

### UI

- 共有シート内に「QRで共有」を表示する。
- 押下すると招待リンクのQRコードを表示する。
- 近くにいる相手はスマホで読み取り、ログイン後に参加する。

### 実装方針

- QRは招待リンクのURLを元に生成する。
- 依存を増やしたくない場合は、まずは外部ライブラリなしで「リンクコピー」を優先する。
- QRを入れる場合は小さなQR生成ライブラリを使うか、Canvas/SVG生成を検討する。

### 受け入れ条件

- QRコードを読み取ると `/invite/[token]` が開く。
- ログイン済みなら共有メンバーに追加される。
- 未ログインならログイン後に参加できる。
- QRは公開閲覧リンクではなく招待リンクを表す。

## 推奨実装順

1. 案2: 共有リスト画面に「人を追加」ボタンを追加する。
2. 案3: 招待リンクのDB/API/画面を追加する。
3. 案5: 招待リンクをQR表示できるようにする。

## API案

- `POST /api/lists/[id]/invite`
  - 所有者だけ実行可能。
  - 招待トークンを発行または再利用する。
  - `{ inviteUrl }` を返す。

- `POST /api/invite/[token]/accept`
  - ログイン必須。
  - 対象リストに `editor` として追加する。
  - `{ listId }` を返す。

## テスト観点

- 所有者はリスト画面からメールで共有できる。
- ゲストは共有操作時にログイン導線が表示される。
- 所有者は招待リンクをコピーできる。
- 招待リンクからログインユーザーが参加できる。
- 参加済みユーザーがリンクを開いても重複しない。
- 無効な招待リンクはエラーになる。
- QR読み取り先が招待リンクになっている。
